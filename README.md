# QAOA for MaxCut on Cyclic Graphs
QAOA for MaxCut on cyclic graphs | Testing the hypothesis that convergence requires n/2 layers | Comparison of 4 optimizers (Adagrad, Adam, GD, COBYLA)

# Описание 

В данном проекте исследуется применение квантового алгоритма QAOA (Quantum Approximate Optimization Algorithm) для решения задачи MaxCut на циклических графах.

**Основная цель работы:** эмпирически проверить гипотезу о том, что для сходимости QAOA на циклическом графе с n вершинами требуется p = n/2 слоев. Дополнительно проводится сравнительный анализ эффективности четырех классических оптимизаторов — Adagrad, Adam, классического градиентного спуска (GD) и безградиентного метода COBYLA — при подборе параметров $`γ`$ и $`β`$ в гибридном квантово-классическом цикле.

Эксперименты проводятся для циклических графов с числом вершин n от 4 до 12. Для каждого графа QAOA запускается с последовательным увеличением числа слоев p до достижения сходимости. Критерием сходимости служит достижение целевой энергии $`E_{target} = n`$ для четных n или $`E_{target} = n-1`$ для нечетных n с точностью $`ε = 0.1`$. Для каждого значения p параметры $`γ`$ и $`β`$ оптимизируются всеми четырьмя методами, после чего сравниваются скорость сходимости и качество полученного решения.  

<br>  

## Постановка задачи MaxCut
Дан граф G = (V, E) с числом вершин V и числом рёбер E. Требуется разбить вершины графа на два множества таким образом, чтобы количество рёбер, соединяющих вершины из разных множеств, было максимальным.

Функция стоимости для MaxCut имеет вид:

```math
C = \frac{1}{2} \sum_{(i,j) \in E} (1 - z_i z_j)
```

где $`z_i`$ и $`z_j`$ — бинарные переменные, ассоциированные с вершинами i и j, которые принимают значения +1 или -1 в зависимости от того, в какое из двух множеств, определяемых разрезом, они попадают.
В текущей реализации QAOA для MaxCut на циклическом графе используется эквивалентный гамильтониан $`H_C = \sum_{(i,j) \in E} Z_i Z_j`$, который с точностью до константы и масштаба соответствует минимизации функции -C.

<br>

Квантовая схема состоит из трех основных этапов:  

`Инициализация ──► p слоев QAOA ──► Измерение`

### Инициализация квантового состояния
   
Первый шаг алгоритма — подготовка равномерной суперпозиции всех возможных битовых строк. Для этого к каждому кубиту, исходно находящемуся в состоянии ∣0⟩, применяется **гейт Адамара H**:

```math
H|0\rangle = \frac{|0\rangle + |1\rangle}{\sqrt{2}} = |+\rangle
```
В результате для системы из n кубитов получается состояние $`|\psi_0\rangle`$, представляющее собой равномерную суперпозицию всех $`2^n`$ битовых строк: 

```math
|\psi_0\rangle = H^{\otimes n} |0\rangle^{\otimes n} = \frac{1}{\sqrt{2^n}} \sum_{x \in \{0,1\}^n} |x\rangle
```

Таким образом, при запуске алгоритма все возможные решения задачи MaxCut представлены в квантовом состоянии с одинаковой амплитудой $`\frac{1}{\sqrt{2^n}}`$ и, соответственно, на входе имеют равную вероятность появления.

<br>  

### Слои QAOA

Ключевой элемент QAOA — построение параметризованной квантовой схемы на основе двух гамильтонианов - целевого (cost Hamiltonian) и смешивающего (mixer Hamiltonian). Целевой гамильтониан $`ℋ_C`$ кодирует решаемую задачу: его основное состояние (с минимальной энергией) соответствует искомому оптимальному решению и представляется оператором эволюции вида:

```math
U(\gamma, \mathcal{H}_C) = \prod_{(i,j)} e^{-i\gamma Z^{(i)}Z^{(j)}}
```
где вариационный вектор параметров $`\boldsymbol{\gamma} = (\gamma_1, \dots, \gamma_p)`$ играет роль времени эволюции.  
Данный слой представлен последовательностью следующих гейтов: `CNOT(i,j) → RZ(γ, j) → CNOT(i,j)`  


Смешивающий слой строится из операторов Паули X:

```math
U(\beta, B) = \prod_{j=1}^n e^{i\beta X^{(j)}}
```
со вторым вектором параметров $\boldsymbol{\beta} = (\beta_1, \dots, \beta_p)`$.  

Эти два слоя поочерёдно повторяются p раз, образуя полный анзац  

```math
|\psi(\boldsymbol{\gamma},\boldsymbol{\beta})\rangle = \prod_{k=1}^p U_B(\beta_k) U_C(\gamma_k) |+\rangle^{\otimes n}
```

Классический оптимизатор подбирает параметры $`γ`$ и $`β`$ для каждого слоя, минимизируя ожидаемое значение $`⟨ℋ_C⟩`$. В результате измерений на выходе схемы битовые строки с наибольшей вероятностью соответствуют приближённым решениям оптимизационной задачи. 

<br>  

## Сходимость QAOA 

С увеличением числа слоев минимальное среднее значение $`E_p(γ, β)`$ гамильтониана $`ℋ_C`$, найденное с помощью алгоритма QAOA, стремится к минимальному значению $`min_z`$ $`ℋ_C(z)`$ среди всех возможных битовых строк z:

```math
\lim_{p \to \infty} \min_{\boldsymbol{\gamma},\boldsymbol{\beta}} E_p(\boldsymbol{\gamma},\boldsymbol{\beta}) = \min_z ℋ_C(z)
```
где $`E_p(γ,β) = ⟨ψ(γ,β)|ℋ_C|ψ(γ,β)⟩`$ — средняя энергия.  

Поскольку на практике число слоев конечно, алгоритм дает лишь приближенное решение. 

<br>  

## Сравнение оптимизаторов

В ходе экспериментов для графов с числом вершин n = 4-12 были протестированы четыре оптимизатора: Adagrad, Adam, GradientDescent и COBYLA.

1. **Адаптивные оптимизаторы (Adagrad, Adam) показывают стабильную сходимость**
   - Достигают целевой энергии при p = n/2 (для чётных n) или p = (n-1)/2 (для нечётных n)
   - Сходятся за 40-50 итераций независимо от размера графа
   - Демонстрируют монотонное улучшение целевой функции

2. **GradientDescent (стандартный градиентный спуск) неэффективен**
   - Застревает в локальных минимумах уже при n ≥ 5
   - Финальная энергия часто значительно ниже целевой (что указывает на некорректную сходимость)
   - Чувствителен к выбору шага обучения

3. **COBYLA (безградиентный метод) работает только для малых n**
   - При n ≤ 5 достигает сходимости в ряде случаев, однако требует большее число слоёв 
   - При n ≥ 6 перестаёт сходиться к целевой энергии

- Надёжную сходимость
- Умеренное число итераций (≈50)
